# スライドの作り方

- いきなりスライドを作らない
- 先にストーリーをメモ帳等で作成する

----

# 心構え
- スライドは伝える手段のひとつ。
- 基本的にはトークが中心
- スライドは印象的、効果的に伝えるアイテム
  
----

# スライドの種類
 
- 高橋メソッド  文字
- ゴーディンメソッド 画像
- レッシグメソッド 文字と画像

----

# 注意事項

- スライドをカンペにしない
	- パトスが感じられない、スライド渡せばいいやん。
- アニメーションの多様
	- ガチャガチャ動くと記憶に残らない
- フォントサイズは統一
	- 1〜2種類。大抵の場合ばメイリオでOK。数字は別にするとかはあり

----

- フォーマットトーンを合わせる
	- 全体に統一感
- 文字はヘッダ、フッターに被らせない
- フォントサイズは20pt 以上
	- これ以上小さいと見えずらくなる（箱のサイズが大きければもっと大きくする方が良い）
- 印刷用とプレゼン用は分ける

----

- 話したいことは書かない
	- カンペにしないというのと同じ
- 見える化する。
	- 数字を入れる等をして、客観性、説得力あるものとする。ロゴスにも繋がる

----

- トークを止め無い
	- もらっている時間の中は喋り続ける。
	- 時間を大切に
		- 参加している人 * 単価は意識する
- 必ずバックアップを用意
	- デモでうまくいか無い時の事を考える
	- 大事な場面ではPC2台持ちもあり。逃げで動画を用意しておくのもあり



----

## デモンストレーションを行う場合

- デモ用にシナリオを作成
	- キチンと見えるように。コンソールでポチポチでは後ろの方は見えない。windowsであれば拡大鏡や、ズーミットといったソフトウェアを使う
	- ムービーも同様に、見えないと意味が無い。

# 6. 本番中

----

## 意識する事
- 挨拶をしっかりする
- 時間を守る
- スライドを読まない、カンペにしない
- カンペが必要な場合は別途用意する
- つかみ。大事
- さぐり？
	- 場をあっためる。質問して、ユーザーを引きつける

----

- つなぎ。
	- 前の人のプレゼンを受けて、序盤を繋ぐ等。全体通して一体感がある等の印象を与える事ができる
- 経験を話す。
	- 仕様だけでなく、具体的な話
- 等身大で話す
	- 同じ目線で話す

----

- 一定のトーンで話さない
	- スライドだけ見て話すと、同じトーンになってしまう。同じトーンだと眠くなる。
	- 比較を話す時は、違いを話す。
		- 相手をけなさずに、こちらの良さを話す。相手をけなすとイメージが悪い。

----

- 口癖
	- えー、とか、あーとかは直すのは難しい。 耳障りにならない程度にすること。
	- 直す場合は、自分の録音を聴いて耳障りなものが無いか確認して、意識する。
- ペースメーカーを見つける
	- 頷いてくれる人を探す。ふんふんしてくれれば進めやすい

----

- 視点誘導
	- トーク等で自分を見て欲しい時は、スライドをブラックアウト「スライドをけす」とよい。
- 汚い言葉をさける
	- みえにくい →みえずらち
	- わかりにくい → わかりづらい
	- 汚い → 美しく無い

----

- 言葉を修飾する
	- 今日お集まりのみなさん → 今日お忙しい中お集まりのみなさん


----

##  スライドを送る前に喋り始める。
   
----
